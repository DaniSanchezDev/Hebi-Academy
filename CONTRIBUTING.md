# Guía de Contribución

¡Gracias por tu interés en contribuir a Hebi Academy! Aquí encontrarás toda la información necesaria para colaborar en el proyecto.

## 🚀 Cómo empezar

1. Haz un fork del repositorio
2. Crea una rama para tu característica o corrección:
   ```bash
   git checkout -b feature/nombre-de-tu-caracteristica
   # o
   git checkout -b fix/nombre-de-tu-correccion
   ```
3. Haz commit de tus cambios con mensajes descriptivos
4. Haz push a tu rama: `git push origin nombre-de-tu-rama`
5. Abre un Pull Request

## 📝 Estándares de Código

- Usa TypeScript en todo el proyecto
- Sigue las convenciones de nomenclatura de React/Next.js
- Mantén el código limpio y bien documentado
- Asegúrate de que tu código pase el linter y los tests

## 🐛 Reporte de Errores

Por favor, usa el sistema de issues de GitHub para reportar errores. Incluye:

- Descripción detallada del error
- Pasos para reproducirlo
- Comportamiento esperado vs. real
- Capturas de pantalla (si aplica)
- Versión del proyecto y entorno

## 💡 Solicitud de Características

Abre un issue describiendo la característica que te gustaría ver implementada. Incluye:

- Descripción detallada
- Casos de uso
- Beneficios esperados

## 📦 Proceso de Revisión

1. Crea un Pull Request con tus cambios
2. Asegúrate de que todos los tests pasen
3. Actualiza la documentación si es necesario
4. Un mantenedor revisará tu PR y te dará feedback

## 📄 Licencia

Al contribuir, aceptas que tus contribuciones estarán bajo la licencia MIT.
